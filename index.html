P10_resize方法用法

发表于 2023-10-28 分类于 OpenCV

图像的放大与缩小

  • resize(src, dsize[, fx[,fy[, interpolation]]])

    • src：要缩放的图片
    • dsize：缩放后的图片大小，元组和列表表示均可.
    • dst：可选参数，缩放之后的输出图片.
    • fx, yx：x轴和y轴的缩放比，即高度和宽度的缩放比.
    • interpolation：插值算法，只要有以下几种：
      • INTER_NEAREST：邻近插值，速度快，效果差.
      • INTER_LINEAR：双线性插值，使用原图中的4个点进行插值，默认为此选项.
      • INTER_CUBIC：三次插值，原图中的16个点.
      • INTER_AREA：区域插值，效果最好，计算时间最长.

P9_图像的逻辑运算

发表于 2023-10-28 分类于 OpenCV

图像的逻辑运算

非运算

  • 与Python原生的非运算不同
  • 图像非运算，用255减每个像素的值，底片效果

与运算

  • 图像之间与运算：
  • 对应位置像素进行与运算，两张图片尺寸通道需相同.
  • 两个数相与，结果通常会变小，255&255的结果是255，所以图片的与运算结果不会超过255

或运算

  • 图像之间或运算：
  • 对应位置像素进行或运算，两张图片的尺寸通道需相同.
  • 或运是与运算的相反运算，结果相比原图会更亮

异或运算

  • 图像之间运算
  • 对应位置进行异或运算，两张图片的尺寸通道需相同.

P7_图像的算数运算

发表于 2023-10-28 分类于 OpenCV

图像的加减乘除运算

图像的加运算

图像与图像相加

  • 图片与图片相加：对应像素的值相加.
  • 两种方法，使用+运算符或使用cv2.add()函数执行图像的加法运算
  • 运算规则：
    • 使用+运算符：超出255的数字会被截断（相当于%255或-255）.
    • 使用cv2.add()：超过255的值全部设为255.
  • 加法操作要求两个图片的长宽相同，通道数相同.
图像与一个值相加
  • 图像与单个数相加：每个像素分别与这个值相加.
  • 只能使用+运算符，cv2.add()函数不适用此类运算.
  • 运算规则：同上使用+运算符，超出255的数字会被截断（相当于%255或-255）

图像的减乘除

  • 所有运算规则同加法

图像的减法运算

  • 图像减图像，可以使用cv2.subtract()函数或-运算符，规则同加法
  • 图像减一个数，只能使用运算符，规则同加法

图像的乘法运算

  • 图像乘图像，可以使用cv2.multiply()函数或*运算符，规则同加法
  • 图像乘一个数，只能使用运算符，规则同加法

图像的除法运算

  • 图像除图像，可以使用cv2.divede()函数或/运算符，规则同加法
  • 图像除一个数，只能使用运算符，规则同加法

P6_绘制图形与文字

发表于 2023-10-27 更新于 2023-10-28 分类于 OpenCV

利用OpenCv提供的绘制图形api可以轻松的在图像上绘制各种图形，比如直线，矩形，圆，椭圆等图形。

直线和矩形

  • line(img, pt1, pt2, color, thickness, lineType, shift)

    • img：在哪个图像上画线
    • pt1, pt2：开始点，结束点（元组）指定线的开始与结束位置
    • color：颜色
    • thickness：线宽
    • lineType：线型，线型为-1, 4, 6, 16,默认为8
    • shift：坐标缩放比例

  • rectangle(img, pt1, pt2, color, thickness lineType, shift)

    • 参数与line()函数一致

圆形

  • circle(img, center, radius, color, thickness, lineType, shift)
    • img：在哪个图像上画圆
    • center：中心点（元组）
    • radius：半径
    • color：颜色
    • thickness 线宽
    • lineType：线型，线型为-1, 4, 6, 16,默认为8
    • shift：坐标缩放比例

椭圆

  • ellipse(img, 中心点, 长宽的一半, 角度, 从哪个角度开始，从哪个角度结束, …)

多边形

  • polylines(img, pts, isClosed, color, thickness, lineType bottomLeftOrigin)
    • img：绘制在那张图像上
    • pts：多边形角点集合(元组或列表, int32)
    • isClosed：多边形是否闭合
    • color：颜色
    • thickness：线宽
    • lineType：线型
  • 填充多边形：fillPoly()函数
  • 参数与polylines相同

文本

  • putText(img, text, org, fontFace, fontScale, color, thinkness, lineType, bottomLeftOrigin)

    • text：要绘制的文本
    • org：文本在图片中的左下角坐标
    • fontFace：字体类型即字体
    • fontScale：字体大小

  • 绘制中文文本：

  • opencv无法绘制中文文本，需要借助pillow包的功能实现绘制中文字体

HEXO博客概述：

  • hexo：基于Node.js的静态博客框架
  • Node.js：hexo依赖的运行环境
  • git：将hexo生成的博客部署至github的工具
  • nvm：windows系统下的node.js环境版本管理器

如何从零搭建：

1.安装git软件:

  • 官网直接下载最新版安装

2.安装Node.js运行环境：

  • 安装nvm https://github.com/coreybutler/nvm-windows 下载Releases安装
  • 在nvm中安装指定Node.js版本并启用 （大多数情况下最新版即可，有时候可能hexo版本不稳定可以尝试老版本的node环境）
  1. 安装hexo
    • 安装好node环境后直接在cmd中全局安装hexo(hexo相当于安装在当前的Node环境中,使用nvm切换另一版本则需要重新安装)

部署：

  • 在博客目录下右键选择Open Git Bash here开启git终端
  • 输入hexo -v检查hexo是否正常安装
  • 配置文件：
    • 配置文件位于博客文件夹目录下,文件名为_config.yml，一般用于设置部署到的仓库位置

两种不同部署方式：

  • 使用ssh+SSH Key方式部署：

    • 启动git终端，生成SSH Key

      一路回车即可

    • cd至sshkey所在目录.ssh

    • 查看生成的密钥文件并复制内容

    • 在github网站的是指中添加新建的SSH Key即可

    • 最后打开配置文件，将部署方式设为Git并填上ssh的仓库地址

  • 使用https+Token方式部署：
    • 在github网站的Developer Settings中选择Personal access tokens中生成Token
    • 打开配置文件，将部署方式设置为Git,部署仓库地址改为https地址，并按格式在地址前加上token
    • 如果部署时弹出github登录窗口，就选择token并填入。如果只能填账号密码就在密码栏填入token。（github已经关闭了账号密码登录，这里填密码是无法连接到github的）
