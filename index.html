P20_拉普拉斯算子

发表于 2023-10-30 分类于 OpenCV

拉普拉斯算子

  • 索贝尔算子是模拟一阶求导，导数越大的地方说明变化越剧烈，越有可能是边缘.

  • 那如果继续对$f’(t)$求导呢？

  • 我们可以发现“边缘处的二阶导数=0”，可以利用这一特性去寻找图像的边缘.注意有一个问题，二阶求导的0的位置也可能是无意义的位置.

  • 拉普拉斯算子的推导过程：

    • 以x方向求解为例：
      一阶差分：$f’(x) = f(x) - f(x-1)$
      二阶差分：$f’’(x) = f’(x+1) - f’(x) = (f(x+1)-f(x))-(f(x)-f(x-1))$
      化简后：$f’’(y)=(f(y-1)-2f(y))+f(y+1)$

      把x,y方向的梯度叠加在一起.
      $f’’(x, y) = f’_x(x, y)+f’_y(x,y)$
      $f’’(x, y) = f(x-1, y)-2f(x, y)+f(x+1,y)+f(x,y-1)-2f(x,y)+f(x, y+1)$
      $f’’(x, y) = f(x-1, y)+f(x+1,y)+f(x,y-1)+f(x,y+1)-4f(x,y)$

      这个等式可以写成:
      $$
      f’’(x,y) = \begin{bmatrix}
      0 & 1 & 0 \newline
      1 & -4 & 1 \newline
      0 & 1 & 0 \end{bmatrix}
      \odot
      \begin{bmatrix}
      f(x-1, y-1) & f(x, y-1) & f(x+1, y-1) \newline
      f(x-1, y) & f(x, y) & f(x + 1, y) \newline
      f(x-1, y+1) & f(x,y+1) & f(x+1, y+1) \end{bmatrix}
      $$
      这样就得到了拉普拉斯算子的卷积核即卷积模板

P19_沙尔(Scharr)算子

发表于 2023-10-30 分类于 OpenCV

沙尔(Scharr)算子

  • Scharr(src, ddepth, dx, dy[, dst[, scale[, delta[, borderType]]]])
  • 当内核大小为3时，以上Sobel内核可能产生比较明显的误差（毕竟Sobel算子只是求取了导数的近似值）。为了解决这一问题，OpenCV提供了Scharr函数，但该函数仅作用于大小为3的内核。该函数的运算与Sobel函数一样快，但结果更加精确.
  • Scharr算子和Sobel很类似，只不过使用了不同的Kernel值，放大了像素变化的情况：
  • Scharr算子的特点：
    • 只支持3*3大小的kernel，所以没有kernel参数.
    • 只能求x方向或y方向的边缘.
    • 把Sobel算子的ksize设为-1就是Scharr算子.
    • Scharr算子擅长寻找细小的边缘，一般用的较少.

P18_索贝尔(sobel)算子

发表于 2023-10-29 更新于 2023-10-30 分类于 OpenCV

索贝尔(sobel)算子

  • 前面讲的各种滤波主要功能都是去除噪声，这样的滤波统称为高通滤波；而接下里要将的各种算子，主要是用来寻找边缘，统称为低通滤波.
  • 边缘是像素值发生跃迁的位置，是图像的显著特征之一，在图像特征提取，对象检测，模式识别等方面都有重要的作用.
  • 人眼如何识别边缘？
    • 比如有一幅图，图里面有一条线，左边很亮，右边很暗，那人眼就容易识别这条线作为边缘.也就是像素的灰度值快速变化的地方.
  • sobel算子对于图像求一阶导数。一阶导数越大，说明像素在该方向的变化越大，边缘信号越强。
  • 因为图像的灰度值都是离散的数字，sobel算子采用离散差分算子计算图像像素点值的近似梯度.
  • 图像是二维的，即沿着宽度/高度两个方向.
  • 我们使用两个卷积核对原图像进行处理：
    • 水平方向
    • 垂直方向
  • 这样，我们就得到了两个新的矩阵，分别反映了每一点像素在水平方向的亮度变化情况和在垂直方向上的亮度变化情况.
  • 综合考虑这两个反向的变化，我们使用以下公式反应某个像素的梯度变化情况.
    $$
    G = \sqrt{G_x^2 + G_y^2}
    $$
  • 有时候为了简单起见，也直接用绝对值相加替代：$G = |G_x|+|G_y|$

P17_双边滤波

发表于 2023-10-29 分类于 OpenCV

双边滤波

  • 双边滤波对于图像的边缘信息能够更好的保存；其原理为一个空间距离相关的高斯函数与一个灰度距离相关的高斯函数相乘.
    • 空间距离：值得是当前点与中心点的欧氏距离.
    • 灰度距离：值得是当前点与中心点灰度的差的绝对值.
  • 双边滤波本质上还是高斯滤波，双边滤波和高斯滤波不同搞得就是：双边滤波既利用了位置信息又利用了像素信息来定义滤波窗口的权重。而高斯滤波只占用了位置信息。
  • 双边滤波可以保留边缘，同时可以对边缘内的区域进行平滑处理.
  • 双边滤波的作用就相当于做了美颜.
    • bilateralFilteer(src, d, sigmaColor, sigmaSpace[, dst[, borderType]])
      • sigmaColor 是计算像素信息使用的sigma
      • sigmaSpace 是计算空间信息使用的sigma

P16_中值滤波

发表于 2023-10-29 分类于 OpenCV

中值滤波

  • 中值滤波的原理非常简单，假设有一个数组[1,5,5,6,7,8,9],取其中的中间值（即中位数），也就是6作为卷积后的结果即可.中值滤波对胡椒噪音（也叫椒盐噪音）效果明显.
  • medianBlur(src, ksize[, dst])
  • 中值滤波的ksize参数为一个整数，若传入元组会报错

方盒滤波

  • boxFilter(src, ddepth, ksize[, dst[, anchor[, normalize[, borderType]]]])

P13_卷积操作

发表于 2023-10-29 分类于 OpenCV

卷积

什么是图片卷积

  • 图片卷积就是卷积核在图像上按行滑动遍历像素时不断地相乘求和的过程

步长

  • 步长就是卷积核在图像上移动的步幅，为了充分扫描图片，步长一般设为1

padding

  • 经过卷积之后的图片的长宽会变小，如果要保持图片大小不变，我们更需要在图片周围填充0；padding值得就是填充0的圈数
  • 我们可以通过公式计算出需要填充0的圈数.
    • 输入体积大小$H_1 * W_1 * D_1$
    • 四个超参数：
      • Filter数量K
      • Filter大小F
      • 步长S
      • 零填充大小P
    • 输出体积大小$H_2 * W_2 * D_2$
      • $H_2 = (H_1 - F + 2P)/S +1$
      • $W_2 = (W_1 - F + 2P)/S +1$
      • $D_2 = K$
  • 如果要保持卷积之后图片大小不变，可以得出等式：$(N + 2P -F +1) = N$，从而推导出$P = \frac{F - 1}{2}$

卷积核的大小

  • 图片卷积中，卷积核的大小一般为奇数，比如33, 55, 7*7.为什么是奇数呢，出于以下两个方面的考虑：
    1. 根据上面padding的计算公式，如果要保持图片大小不变，采用偶数卷积核的话，比如4*4，将会出现填充1.5圈0的情况.
    2. 奇数维度的过滤器有中心，便于指示出过滤器的位置，即OpenCV卷积核的锚点.

卷积的使用示例

  • filter2D(src, ddepth, kernel[, dst[, anchor[, delta[, borderType]]]])
    • ddepth 是卷积之后图片的位深，即卷积之后图片的数据类型，一般设为-1，表示和原题类型一致
    • kernel 是卷积核大小，用元组或者ndarray表示，要求数据类型必须是float型.
    • anchor 锚点，即卷积核的中心点，是可选参数，默认是(-1, -1)
    • delta 可选参数，表示卷积之后额外加的一个值，相当于线性方程中的偏差，默认为0.
    • borderType 边界类型，一般不设.

P12_仿射变换与透视变换

发表于 2023-10-29 更新于 2023-10-30 分类于 OpenCV

放射变换简介

  • 放射变化是图像旋转、缩放、平移的总成，具体的做法是通过一个矩阵和原图片坐标进行计算，得到新的坐标，完成变换.所以杆件就是这个矩阵.
  • warpAffine(src, M, dsize, flags, mode, value)
    • M：变换矩阵
    • dsize：输出图片大小
    • flag：与resize中的插值算法一致
    • mode：边界外推法标志
    • value：填充边界值

图像平移操作

  • 平移矩阵
    • 矩阵中的每个像素由$(x, y)$表示这个像素的坐标。假设沿$x$轴平移$t_x$,沿$y$轴平移$t_y$,那么最后得到的坐标为$(\hat{x},\hat{y})=(x+t_x,y+t_x)$,用矩阵表示就是：
      $$
      \begin{pmatrix}
      \hat{x} \newline
      \hat{y} \newline
      1
      \end{pmatrix} =
      \begin{pmatrix}
      1 & 0 & t_x \newline
      0 & 1 & t_y \newline
      0 & 0 & 1 \newline
      \end{pmatrix}
      \begin{pmatrix}
      x \newline
      y \newline
      1 \newline
      \end{pmatrix}
      $$

图像旋转操作与获取变换矩阵

  • 通过opencv提供的api函数，可以直接获取计算好的变换矩阵
  • 这里获取旋转操作的变换矩阵.
  • getRotationMatrix2D(center, angle, scale)
    • center 中心点，以图片的那个店作为旋转时的中心点.
    • angle 角度，旋转的角度，按照逆时针旋转.
    • scale 缩放比例，先把图片进行什么样的缩放.

三点变换

  • getAffineTransform(src[], dst[]) 通过三点可以确定变换后的位置，相当于解方程，3个点对应3个方程，能解出偏移的参数和旋转的角度.
    • src原目标的三个点
    • dst对应变换后的三个点

透视变换

  • 透视变换就是江一中坐标系变换为另一种坐标系.简单来说就是可以把一张“斜”的图片变“正”.
  • warpPersoective(img, M, dsize, …)
  • 对于透视变换来说，M是一个3*3的矩阵.
  • getPerspectiveTransform(src, dst) 获取透视变换的变换矩阵，需要4个点，即图片的4个角

P11_图像的翻转与旋转

发表于 2023-10-29 分类于 OpenCV

图像的翻转

  • flip(src, flipCode)
    • flipCode = 0 表示上下翻转
    • filpCode > 0 表示左右翻转
    • flipCode < 0 上下 + 左右

图像的旋转

  • rotate(img, rotateCode)
  • rotateCode参数只能使用指定的参数选项，不能自己设置角度.
    • ROTAET_90_CLOCKWISE 90度顺时针
    • ROTATE_180 180度
    • ROTATE_90_COUNTERCLOCKWISE 90度逆时针
