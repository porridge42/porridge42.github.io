P13_卷积操作

发表于 2023-10-29 分类于 OpenCV

卷积

什么是图片卷积

  • 图片卷积就是卷积核在图像上按行滑动遍历像素时不断地相乘求和的过程

步长

  • 步长就是卷积核在图像上移动的步幅，为了充分扫描图片，步长一般设为1

padding

  • 经过卷积之后的图片的长宽会变小，如果要保持图片大小不变，我们更需要在图片周围填充0；padding值得就是填充0的圈数
  • 我们可以通过公式计算出需要填充0的圈数.
    • 输入体积大小$H_1 * W_1 * D_1$
    • 四个超参数：
      • Filter数量K
      • Filter大小F
      • 步长S
      • 零填充大小P
    • 输出体积大小$H_2 * W_2 * D_2$
      • $H_2 = (H_1 - F + 2P)/S +1$
      • $W_2 = (W_1 - F + 2P)/S +1$
      • $D_2 = K$
  • 如果要保持卷积之后图片大小不变，可以得出等式：$(N + 2P -F +1) = N$，从而推导出$P = \frac{F - 1}{2}$

卷积核的大小

  • 图片卷积中，卷积核的大小一般为奇数，比如33, 55, 7*7.为什么是奇数呢，出于以下两个方面的考虑：
    1. 根据上面padding的计算公式，如果要保持图片大小不变，采用偶数卷积核的话，比如4*4，将会出现填充1.5圈0的情况.
    2. 奇数维度的过滤器有中心，便于指示出过滤器的位置，即OpenCV卷积核的锚点.

卷积的使用示例

  • filter2D(src, ddepth, kernel[, dst[, anchor[, delta[, borderType]]]])
    • ddepth 是卷积之后图片的位深，即卷积之后图片的数据类型，一般设为-1，表示和原题类型一致
    • kernel 是卷积核大小，用元组或者ndarray表示，要求数据类型必须是float型.
    • anchor 锚点，即卷积核的中心点，是可选参数，默认是(-1, -1)
    • delta 可选参数，表示卷积之后额外加的一个值，相当于线性方程中的偏差，默认为0.
    • borderType 边界类型，一般不设.

P12_仿射变换与透视变换

发表于 2023-10-29 更新于 2023-10-30 分类于 OpenCV

放射变换简介

  • 放射变化是图像旋转、缩放、平移的总成，具体的做法是通过一个矩阵和原图片坐标进行计算，得到新的坐标，完成变换.所以杆件就是这个矩阵.
  • warpAffine(src, M, dsize, flags, mode, value)
    • M：变换矩阵
    • dsize：输出图片大小
    • flag：与resize中的插值算法一致
    • mode：边界外推法标志
    • value：填充边界值

图像平移操作

  • 平移矩阵
    • 矩阵中的每个像素由$(x, y)$表示这个像素的坐标。假设沿$x$轴平移$t_x$,沿$y$轴平移$t_y$,那么最后得到的坐标为$(\hat{x},\hat{y})=(x+t_x,y+t_x)$,用矩阵表示就是：
      $$
      \begin{pmatrix}
      \hat{x} \newline
      \hat{y} \newline
      1
      \end{pmatrix} =
      \begin{pmatrix}
      1 & 0 & t_x \newline
      0 & 1 & t_y \newline
      0 & 0 & 1 \newline
      \end{pmatrix}
      \begin{pmatrix}
      x \newline
      y \newline
      1 \newline
      \end{pmatrix}
      $$

图像旋转操作与获取变换矩阵

  • 通过opencv提供的api函数，可以直接获取计算好的变换矩阵
  • 这里获取旋转操作的变换矩阵.
  • getRotationMatrix2D(center, angle, scale)
    • center 中心点，以图片的那个店作为旋转时的中心点.
    • angle 角度，旋转的角度，按照逆时针旋转.
    • scale 缩放比例，先把图片进行什么样的缩放.

三点变换

  • getAffineTransform(src[], dst[]) 通过三点可以确定变换后的位置，相当于解方程，3个点对应3个方程，能解出偏移的参数和旋转的角度.
    • src原目标的三个点
    • dst对应变换后的三个点

透视变换

  • 透视变换就是江一中坐标系变换为另一种坐标系.简单来说就是可以把一张“斜”的图片变“正”.
  • warpPersoective(img, M, dsize, …)
  • 对于透视变换来说，M是一个3*3的矩阵.
  • getPerspectiveTransform(src, dst) 获取透视变换的变换矩阵，需要4个点，即图片的4个角

图像的翻转

  • flip(src, flipCode)
    • flipCode = 0 表示上下翻转
    • filpCode > 0 表示左右翻转
    • flipCode < 0 上下 + 左右

图像的旋转

  • rotate(img, rotateCode)
  • rotateCode参数只能使用指定的参数选项，不能自己设置角度.
    • ROTAET_90_CLOCKWISE 90度顺时针
    • ROTATE_180 180度
    • ROTATE_90_COUNTERCLOCKWISE 90度逆时针

P9_图像的逻辑运算

发表于 2023-10-28 分类于 OpenCV

图像的逻辑运算

非运算

  • 与Python原生的非运算不同
  • 图像非运算，用255减每个像素的值，底片效果

与运算

  • 图像之间与运算：
  • 对应位置像素进行与运算，两张图片尺寸通道需相同.
  • 两个数相与，结果通常会变小，255&255的结果是255，所以图片的与运算结果不会超过255

或运算

  • 图像之间或运算：
  • 对应位置像素进行或运算，两张图片的尺寸通道需相同.
  • 或运是与运算的相反运算，结果相比原图会更亮

异或运算

  • 图像之间运算
  • 对应位置进行异或运算，两张图片的尺寸通道需相同.

P7_图像的算数运算

发表于 2023-10-28 分类于 OpenCV

图像的加减乘除运算

图像的加运算

图像与图像相加

  • 图片与图片相加：对应像素的值相加.
  • 两种方法，使用+运算符或使用cv2.add()函数执行图像的加法运算
  • 运算规则：
    • 使用+运算符：超出255的数字会被截断（相当于%255或-255）.
    • 使用cv2.add()：超过255的值全部设为255.
  • 加法操作要求两个图片的长宽相同，通道数相同.
图像与一个值相加
  • 图像与单个数相加：每个像素分别与这个值相加.
  • 只能使用+运算符，cv2.add()函数不适用此类运算.
  • 运算规则：同上使用+运算符，超出255的数字会被截断（相当于%255或-255）

图像的减乘除

  • 所有运算规则同加法

图像的减法运算

  • 图像减图像，可以使用cv2.subtract()函数或-运算符，规则同加法
  • 图像减一个数，只能使用运算符，规则同加法

图像的乘法运算

  • 图像乘图像，可以使用cv2.multiply()函数或*运算符，规则同加法
  • 图像乘一个数，只能使用运算符，规则同加法

图像的除法运算

  • 图像除图像，可以使用cv2.divede()函数或/运算符，规则同加法
  • 图像除一个数，只能使用运算符，规则同加法

P10_resize方法用法

发表于 2023-10-28 更新于 2023-10-29 分类于 OpenCV

图像的放大与缩小

  • resize(src, dsize[, fx[,fy[, interpolation]]])

    • src：要缩放的图片
    • dsize：缩放后的图片大小，元组和列表表示均可.
    • dst：可选参数，缩放之后的输出图片.
    • fx, yx：x轴和y轴的缩放比，即高度和宽度的缩放比.
    • interpolation：插值算法，只要有以下几种：
      • INTER_NEAREST：邻近插值，速度快，效果差.
      • INTER_LINEAR：双线性插值，使用原图中的4个点进行插值，默认为此选项.
      • INTER_CUBIC：三次插值，原图中的16个点.
      • INTER_AREA：区域插值，效果最好，计算时间最长.

P6_绘制图形与文字

发表于 2023-10-27 更新于 2023-10-28 分类于 OpenCV

利用OpenCv提供的绘制图形api可以轻松的在图像上绘制各种图形，比如直线，矩形，圆，椭圆等图形。

直线和矩形

  • line(img, pt1, pt2, color, thickness, lineType, shift)

    • img：在哪个图像上画线
    • pt1, pt2：开始点，结束点（元组）指定线的开始与结束位置
    • color：颜色
    • thickness：线宽
    • lineType：线型，线型为-1, 4, 6, 16,默认为8
    • shift：坐标缩放比例

  • rectangle(img, pt1, pt2, color, thickness lineType, shift)

    • 参数与line()函数一致

圆形

  • circle(img, center, radius, color, thickness, lineType, shift)
    • img：在哪个图像上画圆
    • center：中心点（元组）
    • radius：半径
    • color：颜色
    • thickness 线宽
    • lineType：线型，线型为-1, 4, 6, 16,默认为8
    • shift：坐标缩放比例

椭圆

  • ellipse(img, 中心点, 长宽的一半, 角度, 从哪个角度开始，从哪个角度结束, …)

多边形

  • polylines(img, pts, isClosed, color, thickness, lineType bottomLeftOrigin)
    • img：绘制在那张图像上
    • pts：多边形角点集合(元组或列表, int32)
    • isClosed：多边形是否闭合
    • color：颜色
    • thickness：线宽
    • lineType：线型
  • 填充多边形：fillPoly()函数
  • 参数与polylines相同

文本

  • putText(img, text, org, fontFace, fontScale, color, thinkness, lineType, bottomLeftOrigin)

    • text：要绘制的文本
    • org：文本在图片中的左下角坐标
    • fontFace：字体类型即字体
    • fontScale：字体大小

  • 绘制中文文本：

  • opencv无法绘制中文文本，需要借助pillow包的功能实现绘制中文字体
