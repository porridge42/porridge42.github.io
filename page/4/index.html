hexo的搭建配置与部署方法备用

发表于 2023-10-23 分类于 工具

HEXO博客概述：

  • hexo：基于Node.js的静态博客框架
  • Node.js：hexo依赖的运行环境
  • git：将hexo生成的博客部署至github的工具
  • nvm：windows系统下的node.js环境版本管理器

如何从零搭建：

1.安装git软件:

  • 官网直接下载最新版安装

2.安装Node.js运行环境：

  • 安装nvm https://github.com/coreybutler/nvm-windows 下载Releases安装
  • 在nvm中安装指定Node.js版本并启用 （大多数情况下最新版即可，有时候可能hexo版本不稳定可以尝试老版本的node环境）
  1. 安装hexo
    • 安装好node环境后直接在cmd中全局安装hexo(hexo相当于安装在当前的Node环境中,使用nvm切换另一版本则需要重新安装)

部署：

  • 在博客目录下右键选择Open Git Bash here开启git终端
  • 输入hexo -v检查hexo是否正常安装
  • 配置文件：
    • 配置文件位于博客文件夹目录下,文件名为_config.yml，一般用于设置部署到的仓库位置

两种不同部署方式：

  • 使用ssh+SSH Key方式部署：

    • 启动git终端，生成SSH Key

      一路回车即可

    • cd至sshkey所在目录.ssh

    • 查看生成的密钥文件并复制内容

    • 在github网站的是指中添加新建的SSH Key即可

    • 最后打开配置文件，将部署方式设为Git并填上ssh的仓库地址

  • 使用https+Token方式部署：
    • 在github网站的Developer Settings中选择Personal access tokens中生成Token
    • 打开配置文件，将部署方式设置为Git,部署仓库地址改为https地址，并按格式在地址前加上token
    • 如果部署时弹出github登录窗口，就选择token并填入。如果只能填账号密码就在密码栏填入token。（github已经关闭了账号密码登录，这里填密码是无法连接到github的）

  • hexo clean清除缓存重新生成部署

P101IO流_对象流

发表于 2023-05-25 更新于 2023-05-29 分类于 JavaSE

对象流: ObjectOutputStream/ObjectInputStream

  • 对象流就是可以将对象写入文件或从文件中读取数据再重建成对象的流.
  • 对象流特点:
    • 增强了缓冲区功能.
    • 增强了读写8中基本数据类型和字符串功能.
    • 增强了读写对象的功能：
      • readObject()从流中读取一个对象.
      • writeObject(Object obj)向流中写入一个对象.

        使用流传输对象的过程称为序列化、反序列化.

序列化

  • 将对象通过对象流写入文件的过程称为序列化.
  • 被序列化的对象必须实现Serializable接口.
    • Serializable接口是一个标记接口,不含有任何属性和方法，仅作标记改类可被序列化.

序列化示例:

反序列化

  • 从文件中读取数据并重建对象的过程称为序列化.

序列化示例:

  • 注：使用ois读取文件时同一个文件只能读取一次，若重复读取则会报java.io.EOFException异常.

对象流注意事项

  1. 序列化类必须要实现Serializable接口.

  2. 序列化类中对象属性（引用类型属性）要求实现Serializable接口.

  3. 实现Serializable接口的类理论是需要声明一个serialVersionUID属性:

    • public class Student implements Serializable {
    private static final long serialVersionUID = 100L;
    private String name;
    private int age;
    ......
}
    • serialVersionUID：序列化版本号ID,保证序列化的类和反序列化的类是同一个类.

  4. 含有transient关键字修饰的类不能被序列化.

    transient：瞬间的，使用该修饰符定义的属性止咳存在于内存之中.

  5. 含有静态（static）属性的类也不能序列化.

  6. 序列化多个对象，可以借助集合实现.
