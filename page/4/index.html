P87集合框架_HashSet使用

发表于 2022-12-13 更新于 2023-02-23 分类于 JavaSE

HashSet实现类:

  • HashSet【重点】:
    • 基于HashCode计算元素存放位置。
    • 当存入元素的哈希码相同时，会调用equals进行确认，如果结果为true,则拒绝后者存入。
    • 存储结构：哈希表(数组+链表+红黑树)


输出显示的元素存储顺序和代码中添加时的顺序不一致，说明集合中的元素是无序存储的，并无法添加重复的元素

发表于 2022-12-08 分类于 JavaSE

Set 子接口

  • 特点：无序、无下标、元素不可重复。

  • 方法：全部继承自Collection中的方法，并未增加方法。


  • Set接口的实现类:

    • HashSet(重点)：
      • 基于HashCode实现元素不重复。
      • 但存入元素的哈希码表时，会调用equals进行确认，如果结果为true，则拒绝后者存入。
    • TreeSet：
      • 基于排列顺序实现元素不重复。

泛型集合

  • 概念：参数化类型、烈性安全的集合，强制集合元素的类型必须一致。


  • 特点：

    • 编译时即可检查，而非运行时抛出异常。
    • 访问时，不必类型转换（拆箱）。
    • 不同泛型之间引用不能相互赋值，泛型不存在多态。

P83集合框架_泛型接口

发表于 2022-12-07 分类于 JavaSE

泛型接口

  • 泛型接口的使用方法(两种)：
    1. 在定义接口时使用泛型定义，当编写实现类来实现接口中的抽象方法时可以按需求传入不同的类型来重写不同的实现方法。
    2. 编写实现类时使用泛型类，此时接口的泛型类型就为实现类实例化时传入的泛型类型。
  • 定义语法:接口名<T>，T表示是类型占位符，表示一种引用类型，如果编写多个，可以用逗号隔开。

P82集合框架_泛型类

发表于 2022-12-06 更新于 2022-12-07 分类于 JavaSE

泛型类

  • 泛型类的使用方法:
    • 在定义类的时候使用泛型定义，当生成实例化对象时可以依据不同需求灵活传入不同的类型来生成需要的实例对象。
  • 定义语法:类名<T>，T表示是类型占位符，表示一种引用类型，如果编写多个，可以用逗号隔开。
  • 注意:
    1. 泛型只能使用引用类型。
    2. 不同泛型类型对象之间不能相互复制

P81集合框架_泛型概述

发表于 2022-12-06 分类于 JavaSE

泛型概述:

  • Java泛型是JDK1.5引入的一个新特性，其本质是参数化类型，把类型作为参数传递。


  • 常见形式有泛型类、泛型接口、泛型方法。


  • 语法：

    • <T,...>T称为类型占位符，表示一种引用类型。

  • 好处：

  1. 提高代码的重用性。
  2. 防止类型转换异常，提高代码的安全性。
